Hur fungerar webben?

Webben är ett komplext system av sammankopplade teknologier som arbetar tillsammans för att leverera innehåll till miljardtals användare världen över.

🖥️

Klient-Server

Din webbläsare (klient) skickar förfrågningar till servrar som lagrar webbsidor och svarar med innehåll.

Request → Response
🌐

DNS

Domain Name System översätter läsbara domännamn (google.com) till IP-adresser som datorer kan förstå.

example.com → 192.0.2.1
🔒

HTTP/HTTPS

Protokoll som definierar hur meddelanden skickas mellan klient och server. HTTPS lägger till kryptering för säkerhet.

🔒 Säker kommunikation
📄

HTML

HyperText Markup Language definierar strukturen och innehållet på webbsidor med taggar och element.

Struktur & innehåll
🎨

CSS

Cascading Style Sheets styr hur webbsidor ser ut - färger, typografi, layout och responsiv design.

Design & layout
⚡

JavaScript

Programmeringsspråk som gör webbsidor interaktiva och dynamiska med animationer, validering och uppdateringar.

Interaktivitet & logik

Webbförfrågans resa

Vad händer när du skriver in en URL i webbläsaren?

1

Du skriver URL

Du skriver en adress som example.com i webbläsarens adressfält.

2

DNS-uppslagning

Webbläsaren kontaktar en DNS-server för att översätta domännamnet till en IP-adress (t.ex. 93.184.216.34).

3

TCP-anslutning

En TCP-anslutning etableras mellan din dator och servern via den tre-vägs handskakning.

4

HTTP-förfrågan

Webbläsaren skickar en HTTP GET-förfrågan till servern för att be om webbsidan.

5

Server svarar

Servern bearbetar förfrågan och skickar tillbaka HTML, CSS, JavaScript och andra resurser.

6

Rendering

Webbläsaren tolkar HTML, tillämpar CSS-styling och kör JavaScript för att visa den färdiga sidan.

JavaScript - Interaktivitet

Skapar dynamiskt beteende

// Event listener för knappklick
document.querySelector('#myButton').addEventListener('click', function() {
  // Hämta data från API
  fetch('https://api.example.com/data')
    .then(response => response.json())
    .then(data => {
      // Uppdatera DOM med ny data
      document.querySelector('#content').innerHTML = data.message;
      console.log('Data laddad:', data);
    })
    .catch(error => console.error('Fel:', error));
});

Responsiv design

En webbsida som anpassar sig till alla skärmstorlekar

📱

Mobil

Optimerad för pekskärm och små skärmar (320-480px)

max-width: 480px
📱

Surfplatta

Mellanformat för surfplattor (768-1024px)

max-width: 1024px
💻

Desktop

Full bredd för stora skärmar (1200px+)

max-width: 1920px

CSS Media Queries

/* Mobile First Approach */
.container {
  width: 100%;
  padding: 1rem;
}

/* Tablet */
@media (min-width: 768px) {
  .container {
    max-width: 720px;
    margin: 0 auto;
  }
}

/* Desktop */
@media (min-width: 1024px) {
  .container {
    max-width: 960px;
    padding: 2rem;
  }
}

/* Large Desktop */
@media (min-width: 1280px) {
  .container {
    max-width: 1200px;
  }
}

Prestandaoptimering

Viktiga tekniker för snabbare webbsidor

🖼️

Optimera bilder

Använd moderna format som WebP/AVIF, lazy loading och responsiva bilder med srcset.

📦

Minifiera kod

Ta bort onödiga mellanslag och kommentarer från CSS och JavaScript för mindre filstorlekar.

⚡

Använd caching

Spara resurser lokalt i webbläsaren med Service Workers och HTTP cache-headers.

🚀

CDN och komprimering

Använd Content Delivery Networks och Gzip/Brotli-komprimering för snabbare leverans.
